Modelling Systems & Markets in Python

Two-part project: a cleanly designed Python OOP simulation for hatchery operations, and a finance notebook that analyses stocks and crypto with reproducible EDA.

View repo

Problem

Build a maintainable simulation for quarterly operations and a clear, reproducible analysis pipeline for market data. The aim is robust engineering in Task 1 and trustworthy analytics in Task 2.

Approach

  • Task 1: modular classes for the core entities (hatchery, warehouse, supplier, technicians, fish species). Responsibilities are separated: cash flow and labour, stock and depreciation, vendor pricing, species requirements, and technician wages and specialty logic.
  • Task 2: tidy ingest, cleaning, feature engineering, and exploratory plots. Compared risk and return profiles for equities and crypto using consistent windows and summary statistics.
  • Reproducibility: documented methods, defensive input handling in the simulation loop, and deterministic figure export (size and DPI) from the notebook.

Results

  • Simulation: complete quarterly cycles with hiring and firing, sales within labour and stock constraints, depreciation and restocking, and clear console summaries each cycle.
  • Analytics: clear separation between preprocessing and visuals, producing plots that make asset behaviour and volatility-return trade-offs obvious at a glance.
  • Code quality: class docstrings, single-responsibility methods, and error handling for user input across the main loop.

Tech stack

Python. OOP modules for the simulation, pandas, NumPy, and matplotlib for analysis, and reproducible notebooks for EDA.
