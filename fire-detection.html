Fire Detection Using Synthetic Data

Industry dissertation with SYNOPTIX Ltd. I built a computer vision model to detect indoor fires using a mix of real and synthetic images. The goal was strong accuracy with less real data. I used ResNet‑50 for classification, Grad‑CAM for explanations, and standard ROC/PR evaluation.

View repo View report (PDF)

Problem

Real indoor fire images are hard to collect and label. We wanted to see if synthetic images could replace a large share of the real data while keeping low false alarms and high recall for a CCTV setting.

Approach

  • Trained and fine tuned a ResNet‑50 classifier on different mixes of real and synthetic images.
  • Kept dataset sizes fixed with a set random seed for fair comparison. Used simple [0, 1] scaling.
  • Evaluated with ROC and PR curves, confusion matrices, and per class metrics. Used Grad‑CAM to check if the model focused on flames and smoke.

Results

  • Indoor deployment mix (50% real, 50% synthetic): F1 0.984, MCC 0.972, ROC AUC about 0.999 on the indoor test set.
  • Domain shift: The 50/50 fine tuned model beat real only training when moving from outdoor to indoor images.
  • Data efficiency: The mixed data model reached strong performance with far fewer real images compared with real only training.

Tech stack

Python, PyTorch (ResNet‑50), torchvision, scikit‑learn, matplotlib, Grad‑CAM. Experiments run in Google Colab with saved JSON metrics and plots.
